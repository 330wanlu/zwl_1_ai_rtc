## 6. FastAPI 接口说明

> 本阶段**未新增** `1-chat-ai-agent` FastAPI 接口。  
> 以下为阶段0实际调用的数据中台 / 本项目既有接口，含入参出参与完整数据流示例。

### 6.1 `GET /health`（数据中台）

**作用：** 探活 + DB 状态。

**入参：** 无

**出参：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "status": "ok",
    "db": "ok",
    "api_key_required": false,
    "ai_image_sync_enabled": true,
    "video_clip_sync_enabled": true
  }
}
```

---

### 6.2 `POST /api/v1/media/upload`（数据中台）

**作用：** 上传素材，返回可访问 URL。

**入参（multipart/form-data）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | file | 是 | 图片/视频文件 |
| name | string | 否 | 展示名 |
| tags | string | 否 | 逗号分隔标签 |

**出参示例：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 148,
    "name": "RAG高薪就业示意",
    "media_type": "image",
    "file_path": "2026/07/rag_career_5500bd5e.jpg",
    "mime_type": "image/jpeg",
    "url": "http://localhost:8000/media_files/2026/07/rag_career_5500bd5e.jpg",
    "tags": ["RAG", "就业", "演示"]
  }
}
```

---

### 6.3 `POST /api/v1/knowledge`（数据中台）

**作用：** 创建知识并绑定媒体；`status=published` 后可被 `/ai/query` 命中。

**入参（JSON）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 标题 |
| question | string | 否 | 标准问法 |
| answer | string | 是 | 文本答案 |
| intent | string | 否 | 意图代号 |
| keywords | string[] | 否 | 匹配关键词 |
| status | string | 否 | draft/published/disabled，默认 draft |
| priority | int | 否 | 越大越靠前 |
| media_binds | object[] | 否 | `{media_id, sort_order, caption}` |

**本阶段实际请求：**

```json
{
  "title": "RAG就业与业务价值",
  "question": "学RAG有什么用",
  "answer": "学 RAG 能帮你把企业私有知识接到大模型上……你可以边看图边听我说明它的好处。",
  "intent": "rag_career_value",
  "keywords": ["RAG", "学RAG", "有什么用", "就业", "高薪", "好处", "检索增强"],
  "status": "published",
  "priority": 20,
  "media_binds": [
    {"media_id": 148, "sort_order": 0, "caption": "RAG 就业方向示意"},
    {"media_id": 149, "sort_order": 1, "caption": "RAG 价值讲解"}
  ]
}
```

**出参要点：** `data.id`（如 48）、`data.media` 列表含 url。

---

### 6.4 `POST /api/v1/ai/query`（数据中台 · 核心）

**作用：** 外部客服/演示查询：返回文本答案 + 媒体 URL。

**入参：**

| 字段 | 类型 | 必填 | 默认 | 说明 |
|------|------|------|------|------|
| query | string | 是 | - | 用户问句 |
| intent | string | 否 | null | 意图精确匹配加分 |
| limit | int | 否 | 3 | 1~20 |
| need_media | bool | 否 | true | 是否返回媒体 |

**完整数据流示例（学RAG）：**

```
1) 客户端
   POST /api/v1/ai/query
   {"query":"学RAG有什么用","need_media":true,"limit":3}

2) 中台 match_service
   过滤 status=published
   → 对 question/keywords/title 做包含匹配
   → score += priority
   → 取 topN，挂载 media url

3) 响应
{
  "code": 0,
  "message": "ok",
  "data": {
    "matched": true,
    "items": [
      {
        "knowledge_id": 48,
        "title": "RAG就业与业务价值",
        "answer": "学 RAG 能帮你把企业私有知识接到大模型上，做可靠问答和客服助手。市场上相关岗位需求多、薪资也不错，是落地 AI 应用的一条实用路径。你可以边看图边听我说明它的好处。",
        "score": 45.08,
        "media": [
          {
            "id": 148,
            "type": "image",
            "url": "http://localhost:8000/media_files/2026/07/rag_career_5500bd5e.jpg",
            "name": "RAG高薪就业示意",
            "caption": "RAG 就业方向示意"
          },
          {
            "id": 149,
            "type": "video",
            "url": "http://localhost:8000/media_files/2026/07/rag_benefit_1693f019.mp4",
            "name": "RAG价值讲解视频",
            "caption": "RAG 价值讲解"
          }
        ]
      }
    ]
  }
}

4) 客户端再 GET media.url
   → 200 image/jpeg 或 video/mp4
```

**未命中示例：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {"matched": false, "items": []}
}
```

**鉴权：** 当前中台 `API_KEY` 为空，不校验；若配置了 API_KEY，需请求头 `X-API-Key`。

## 7. 踩坑汇总

| # | 问题 | 原因 | 处理 |
|---|------|------|------|
| 1 | 初始「怎么退货」`matched=false` | 库内演示语料缺失/被覆盖 | 重新执行中台 `scripts/seed_demo.py` |
