## 3. 架构说明

体验层叠加在既有「SSE 文字 + pending 媒体」之上，不改主链路：

```
中台 answer
  → 去 URL/Markdown
  → 最多 N 句（默认 4）
  → 再按字符上限截断
  → SSE → TTS

前端气泡
  → 图：小缩略图，点击放大；加载失败显示文案
  → 视频：缩略区域 + controls + muted（不自动出声）

打断
  → 字幕标「已打断」
  → media 留在气泡
  → 未 pop 的 pending 等 TTL 过期
```

---

## 4. 关键设计取舍

| 取舍 | 选择 | 原因 |
|------|------|------|
| 句数默认 | 4 | 与方案「2～4 句」上限对齐，可用 env 调小 |
| 视频 muted | 默认静音 | 避免与 TTS 抢声；用户可手动开声 |
| 未命中日志 | 去掉「未命中」info | 减少控制台噪音；失败仍黄字提示一次 |
| 打断是否清空 pending | 否 | 已展示应保留；未拉到的靠 TTL，避免误删 |

## 6. FastAPI 接口说明

本阶段**无新增路由**。行为变化：

### 6.1 `POST /api/chat_callback`（体验相关）

中台命中后的口语处理：

| 步骤 | 说明 |
|------|------|
| 去噪 | 去掉 `http(s)://`、Markdown 符号 |
| 句数 | `SPOKEN_TEXT_MAX_SENTENCES`（默认 4） |
| 字数 | `SPOKEN_TEXT_MAX_CHARS`（默认 180） |
| 媒体 | 仍只进 store，不进 SSE |

**入参/出参契约不变**（仍为 OpenAI SSE + `[DONE]`）。

未命中/中台失败：不向用户抛独立错误气泡，直接走方舟/RAG。

### 6.2 `GET /api/media/pending` / `health`

契约不变。打断不会主动清空 pending；超时由 `MEDIA_PENDING_TTL_SEC` 清理。

### 6.3 环境变量（新增）

| 变量 | 默认 | 说明 |
|------|------|------|
| `SPOKEN_TEXT_MAX_SENTENCES` | 4 | 口语最多句数 |
