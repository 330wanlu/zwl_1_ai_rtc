## 3. 架构说明

本阶段只落地「可复用模块」，尚未接入语音回调：

```
                    ┌─────────────────────────────┐
  阶段2将接入 ───►  │  chat_callback / sse_stream │
                    └──────────────┬──────────────┘
                                   │ (尚未接线)
          ┌────────────────────────┼────────────────────────┐
          ▼                                                 ▼
┌──────────────────────┐                         ┌──────────────────────┐
│ data_platform_client │  HTTP                   │ media_session_store  │
│ query_ai / health    │ ─────────────────────►  │ push / pop_all       │
└──────────┬───────────┘  /api/v1/ai/query       └──────────┬───────────┘
           │                                                │
           ▼                                                ▼
   6-AI数据中台 :8000                              内存队列 key=room_id
                                           （阶段3 由 GET /api/media/pending 消费）
```

数据流（模块级，当前由自检脚本驱动）：

```
query_ai("学RAG有什么用")
  → AiQueryResult(matched=True, items[0].answer + media[])
  → media_session_store.push("ChatRoom01", ...)
  → pop_all("ChatRoom01") 得到待展示媒体（读后清空）
```

## 4. 关键设计取舍

| 取舍 | 选择 | 原因 |
|------|------|------|
| 客户端同步 vs 异步 | **async httpx** | 与 `main.py` / `knowledge_search` 一致，便于阶段2直接 `await` |
| 失败语义 | 协议成功但未命中 → `AiQueryResult(matched=False)`；网络/超时 → `None` | 区分「无知识」与「中台挂了」，降级策略不同 |
| 缓存实现 | 进程内 dict + threading.Lock | MVP 足够；多进程部署需改 Redis（后续） |
| RoomId | 结构按 room_id 索引，默认 `ChatRoom01` | 符合方案 MVP 方案 A，预留多房间 |
| 空 media 是否 push | **不 push，返回 None** | 避免前端空轮询出无效气泡 |
| 是否本阶段暴露 FastAPI | **否** | pending API 属阶段3；本阶段只交付库模块 |

## 6. FastAPI 接口说明

### 6.1 本阶段新增 FastAPI 路由

**无。** 阶段1仅新增 Python 模块，不注册 HTTP 路由。

> 计划中的 `GET /api/media/pending` 属于**阶段3**。

### 6.2 本模块调用的外部 FastAPI（数据中台）

#### `GET {BASE}/api/v1/ai/health`

| 项 | 说明 |
|----|------|
| 入参 | 无；可选请求头 `X-API-Key` |
| 成功 | `code=0`，client 返回 `data` dict |
| 失败 | client 返回 `None` |

#### `POST {BASE}/api/v1/ai/query`

**client 入参（函数 `query_ai`）：**

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| query | str | 必填 | 用户问句 |
| limit | int | `DATA_PLATFORM_LIMIT`(3) | topN |
| need_media | bool | true | 是否要媒体 |
| intent | str\|None | None | 可选意图 |

**发往中台的 Body 示例：**

```json
{
  "query": "学RAG有什么用",
  "limit": 3,
  "need_media": true
}
```

**完整数据流转示例：**

```
1) await query_ai("学RAG有什么用")
2) HTTP POST http://127.0.0.1:8000/api/v1/ai/query
3) 中台响应：
{
  "code": 0,
  "message": "ok",
  "data": {
    "matched": true,
    "items": [{
      "knowledge_id": 48,
      "title": "RAG就业与业务价值",
      "answer": "学 RAG 能帮你把企业私有知识接到大模型上……",
      "score": 45.08,
      "media": [
        {"id": 148, "type": "image", "url": "http://localhost:8000/media_files/...", "name": "RAG高薪就业示意", "caption": "..."},
        {"id": 149, "type": "video", "url": "http://localhost:8000/media_files/...", "name": "RAG价值讲解视频", "caption": "..."}
      ]
    }]
  }
}
4) 解析为 AiQueryResult；日志：
   [DataPlatform] matched=True items=1 media=2 query='学RAG有什么用'
5) （自检）store.push(... media=...)
6) store.pop_all("ChatRoom01") →
[
  {
    "id": "<uuid>",
    "created_at": 1720000000.0,
    "query": "学RAG有什么用",
    "title": "RAG就业与业务价值",
    "answer": "...",
    "media": [ ... ],
    "knowledge_id": 48,
    "score": 45.08
  }
]
```

**client 返回语义：**

| 情况 | 返回值 |
|------|--------|
| HTTP/超时/JSON/`code!=0` | `None` |
| 成功且命中 | `AiQueryResult(matched=True, items=[...])` |
| 成功但未命中 | `AiQueryResult(matched=False, items=[])` |
| query 空字符串 | `AiQueryResult(matched=False, items=[])`（不发请求） |

### 6.3 媒体缓存模块 API（非 HTTP）

| 方法 | 入参 | 出参 | 说明 |
|------|------|------|------|
| `push(room_id, query, title, answer, media, ...)` | media 需含 url | `PendingMediaItem \| None` | 无有效 media 则 None |
| `pop_all(room_id)` | room_id | `list[dict]` | 弹出并清空 |
| `peek(room_id)` | room_id | `list[dict]` | 调试用只读 |
| `clear(room_id=None)` | 可选 | None | 清空 |
| `stats()` | - | dict | 房间队列计数 |

**环境变量：**

| 变量 | 默认 | 说明 |
|------|------|------|
| `DATA_PLATFORM_ENABLED` | false | 总开关（阶段2才消费） |
| `DATA_PLATFORM_BASE_URL` | http://127.0.0.1:8000 | 中台地址 |
| `DATA_PLATFORM_API_KEY` | 空 | X-API-Key |
| `DATA_PLATFORM_TIMEOUT_MS` | 1500 | 超时毫秒 |
| `DATA_PLATFORM_LIMIT` | 3 | topN |
| `DEFAULT_MEDIA_ROOM_ID` | ChatRoom01 | 默认房间 |
| `MEDIA_PENDING_TTL_SEC` | 120 | TTL 秒 |
| `MEDIA_PENDING_MAX_PER_ROOM` | 5 | 队列上限 |

## 7. 踩坑汇总

| # | 问题 | 处理 |
|---|------|------|
| 1 | Windows 控制台打印中文问句乱码 | 不影响功能；以 JSON 快照 `phase1_self_check.json`（UTF-8）为准 |
| 2 | TTL 单测若 sleep 过短可能偶发不过 | 使用 `ttl_sec=2` + `sleep(2.2)` |
| 3 | `DATA_PLATFORM_ENABLED` 本阶段仍为 false | 刻意为之：避免未改 callback 时产生误解；阶段2接线时再打开 |
