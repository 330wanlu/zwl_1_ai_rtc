## 3. 架构说明

```
用户问句（CustomLLM messages）
        │
        ▼
 POST /api/chat_callback
        │
        ├─ DATA_PLATFORM_ENABLED?
        │     ├─ query_ai(问句) ──► 数据中台 /api/v1/ai/query
        │     │     ├─ 命中：sanitize(answer) → SSE 逐字输出 → [DONE]
        │     │     │         └─ media[] → media_session_store.push(roomId)
        │     │     └─ 未命中/失败 → 降级
        │     └─ 关闭 → 降级
        │
        └─ 降级路径：火山 Viking RAG → 方舟 Ark SSE（或假回复）

前端（阶段4）轮询：
 GET /api/media/pending?roomId=ChatRoom01
        └─ pop_all → 返回 media，读后清空
```

通道分离：

| 通道 | 内容 | 消费者 |
|------|------|--------|
| SSE `delta.content` | 仅口语短文本 | 火山 TTS / 字幕 |
| `GET /api/media/pending` | image/video URL 列表 | 前端 UI（阶段4） |

---

## 4. 关键设计取舍

| 取舍 | 选择 | 原因 |
|------|------|------|
| 命中策略 | S1：直接播中台 answer | MVP 最快；S2（方舟润色）留后续 |
| pending API 是否等阶段3 | **本阶段提前做** | 方案阶段2验收明确要求；否则无法从 HTTP 验证 push |
| RoomId | body.`roomId`/`room_id` 优先，否则 `DEFAULT_MEDIA_ROOM_ID` | 兼容未来扩展；火山默认回调可能不带房间 |
| 空 media 命中 | 仍播文字，不 push | 避免空轮询 |
| SSE 是否可含 URL | **禁止** | 防止 TTS 朗读链接 |

## 6. FastAPI 接口说明

### 6.1 `POST /api/chat_callback`（已有，行为增强）

**作用：** 火山 CustomLLM 回调；现支持中台优先。

**入参（JSON Body，OpenAI 兼容 + 可选扩展）：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| messages | array | 是 | 含 user/system；取最近 user 作为中台 query |
| model | string | 否 | 默认方舟 endpoint |
| stream | bool | 建议 true | 流式 SSE |
| temperature / top_p / max_tokens | number | 否 | 仅降级方舟时使用 |
| roomId / room_id | string | 否 | 媒体挂载房间，默认 `ChatRoom01` |

**出参：** `text/event-stream`，形如：

```text
data: {"id":"...","object":"chat.completion.chunk","choices":[{"delta":{"role":"assistant"}}]}

data: {"choices":[{"delta":{"content":"学"}}]}

data: {"choices":[{"delta":{"content":" R"}}]}
...
data: {"choices":[{"delta":{},"finish_reason":"stop"}],"usage":{...}}

data: [DONE]
```

**完整数据流（命中中台）：**

```
1) POST /api/chat_callback
{
  "messages": [{"role":"user","content":"学RAG有什么用"}],
  "stream": true,
  "roomId": "ChatRoom01"
}

2) 服务端 await query_ai("学RAG有什么用")
   → 数据中台返回 answer + media[image, video]

3) sanitize(answer) → SSE 逐字输出（无 URL）
4) media_session_store.push("ChatRoom01", media=...)
5) 客户端再拉 pending（见下）
```

**完整数据流（未命中）：**

```
1) query_ai → matched=false
2) 火山 RAG 检索（若已配置 AK/SK）
3) 方舟流式回复 → SSE
4) pending 仍为空
```

### 6.2 `GET /api/media/pending`（本阶段新增）

**作用：** 弹出并清空某房间待展示媒体。

**入参（Query）：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roomId | string | 是 | RTC 房间 ID |

**出参：**

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [
      {
        "id": "uuid",
        "created_at": 1720000000.0,

        "title": "RAG就业与业务价值",
        "answer": "学 RAG 能帮你……",
        "knowledge_id": 48,
        "score": 45.08,
        "media": [
          {
            "id": 148,
            "type": "image",
            "url": "http://localhost:8000/media_files/2026/07/rag_career_5500bd5e.jpg",
            "name": "RAG高薪就业示意",
            "caption": "RAG 就业方向示意"
          },
          {
            "id": 149,
```

